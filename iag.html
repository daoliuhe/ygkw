def preprocess(X, y):
    """
    Data preprocessing
        1. data split
        2. data standandization
        
    """

    trainX, testX, trainY, testY = train_test_split(X, y, test_size=0.3, random_state=0, stratify=y)
    print('Training data: %s' % Counter(trainY))
    print('Testing data: %s' % Counter(testY))
    
    s = StandardScaler()
    strain = s.fit_transform(trainX.values)
    stest = s.transform(testX.values)
    dftrain = pd.DataFrame(data=strain, index=trainX.index, columns=trainX.columns)
    dftest = pd.DataFrame(data=stest, index=testX.index, columns=testX.columns)

    return dftrain, dftest, trainY, testY

def evaluation(y_true, y_pred):
    # evaluate prediction result
    
    class_names = ['No Failure', 'Failure']
    conf_mat = confusion_matrix(y_true, y_pred)
    print('Accuracy: %.4f, Recall Score: %.4f' %
          (accuracy_score(y_true, y_pred), recall_score(y_true, y_pred)))
    plot_confusion_matrix(conf_mat, class_names)
    
def plot_confusion_matrix(cm, classes, normalize=False):
    """
    For a confusion matrix C, the element C_ij is equal to the number of observations
    known to be in label i but predicted to be in label j.
    
    """

    if normalize:
        cm = cm.astype('float') / cm.sum(axis=1)[:, np.newaxis]

    pl.imshow(cm, interpolation='nearest', cmap=pl.cm.Blues)
    pl.colorbar()
    tick_marks = np.arange(len(classes))

    fmt = '.2f' if normalize else 'd'
    thresh = cm.max() / 2.0
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        pl.text(j, i, format(cm[i, j], fmt),
                fontsize=11, horizontalalignment="center",
                color="white" if cm[i, j] > thresh else "black")
    pl.xticks(tick_marks, classes, fontsize=11)
    pl.yticks(tick_marks, classes, fontsize=11)
    pl.ylabel('True label', fontsize=13)
    pl.xlabel('Predicted label', fontsize=13)
    pl.tight_layout()

Undersampling for imbalanced classification¶

In previous model, we didn't consider the dataset is imbalanced, which is actually a big issue in classification. In real-world problem, we are more interested in failure events, as they may lead to critical business impact. Therefore, we want to identify as many failures as we can from the whole failure events. In other words, we prefer to increase the recall score, although this may cause a drop in accuracy score.

In this case, undersampling is considered to tackle the imbalance in dataset. The no failure events are under-sampled, and make the positive/negative ratio to be 1.

3. Conclusion¶

In this case study, the failure of widget is investigated. The detection of failure events is considered as a binary classification problem. Random forest is used in the modelling, and the performance is good. Moreover, random forecast is able to rank the importance of input features, which gives a shortlist of potential causes to the failure. In addition, as the dataset is highly imbalanced, undersampling is used to overcome this issue. It is shown that the recall score increases with undersampling.
